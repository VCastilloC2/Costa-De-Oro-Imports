RAG (Retrieval-Augmented Generation) es la arquitectura de inteligencia artificial implementada en Costa de Oro Imports para proporcionar respuestas inteligentes y contextualizadas dentro de la plataforma de e-commerce.

El sistema RAG permite combinar recuperación de información con generación de texto utilizando modelos de lenguaje avanzados (LLM). Gracias a esta arquitectura, la aplicación puede responder preguntas relacionadas con productos, documentación del sistema, políticas de venta y contenido almacenado en la base documental del proyecto.

En Costa de Oro Imports, el módulo RAG fue integrado utilizando Spring Boot y Groq API, permitiendo crear un asistente inteligente capaz de interactuar con los usuarios mediante lenguaje natural.

El flujo general del sistema RAG funciona de la siguiente manera:

1. El usuario realiza una pregunta desde la aplicación web o mediante un endpoint REST.
2. El sistema procesa la pregunta y genera embeddings semánticos.
3. Se realiza una búsqueda de similitud para recuperar los fragmentos de texto más relevantes.
4. Los documentos recuperados se convierten en contexto contextual.
5. El contexto y la pregunta son enviados al modelo LLM.
6. El modelo genera una respuesta basada en información real del sistema.

La principal ventaja de utilizar RAG es reducir respuestas incorrectas o inventadas por el modelo de lenguaje, ya que las respuestas se generan utilizando información previamente almacenada y procesada por el sistema.

La arquitectura RAG implementada en el proyecto está compuesta por varios componentes principales:

- Carga de documentos
- Procesamiento de texto
- Fragmentación de contenido (chunking)
- Recuperación semántica
- Generación de embeddings
- Ranking de similitud
- Generación de respuestas mediante LLM

Los documentos utilizados por el sistema incluyen información académica, documentación técnica y datos relacionados con el funcionamiento de la plataforma Costa de Oro Imports.

El proceso de chunking divide documentos extensos en pequeños fragmentos de texto para mejorar la recuperación semántica y aumentar la precisión de las respuestas generadas.

Durante la recuperación de información se utiliza el concepto de Top-K, que permite seleccionar los fragmentos más relevantes según el nivel de similitud semántica entre la pregunta del usuario y los embeddings almacenados.

El sistema también implementa validaciones y filtros para:

- Evitar respuestas vacías
- Reducir ruido semántico
- Optimizar tiempos de respuesta
- Mejorar la precisión contextual

La integración de inteligencia artificial dentro del proyecto permite modernizar la experiencia de usuario y demostrar la aplicación práctica de tecnologías emergentes dentro de un sistema de comercio electrónico desarrollado con Spring Boot.

¿Qué es RAG?

RAG significa Retrieval-Augmented Generation, que en español puede traducirse como Generación Aumentada por Recuperación. Es una arquitectura utilizada en inteligencia artificial que combina recuperación de información y generación automática de texto mediante modelos de lenguaje (LLM).

El objetivo principal de un sistema RAG es permitir que un modelo de inteligencia artificial responda preguntas utilizando información real obtenida desde documentos, archivos o bases de conocimiento, en lugar de depender únicamente de información aprendida durante el entrenamiento del modelo.

En un sistema tradicional de IA, el modelo genera respuestas basadas únicamente en su conocimiento previo. En cambio, un sistema RAG primero busca información relevante relacionada con la pregunta del usuario y posteriormente utiliza esa información como contexto para generar una respuesta más precisa y contextualizada.

Esta arquitectura es ampliamente utilizada en aplicaciones modernas porque permite:

- Reducir respuestas incorrectas o inventadas
- Utilizar información actualizada
- Consultar documentación privada o empresarial
- Mejorar precisión contextual
- Personalizar respuestas según el dominio del sistema

En Costa de Oro Imports, RAG se utiliza para construir un asistente inteligente capaz de responder preguntas relacionadas con:

- Productos disponibles
- Información académica
- Documentación técnica
- Funcionalidades del sistema
- Arquitectura del proyecto
- Información almacenada en documentos TXT

Gracias a la integración de RAG, la plataforma puede ofrecer respuestas más inteligentes y alineadas con el contenido real del proyecto.
